System Overview

This page demonstrates the conceptual design and data management capabilities of our Vehicle Registration and Licensing System. While this is an HTML-only static demo, it illustrates how a real system would interact with a database for vehicle and owner data, registrations, inspections, insurance, and history tracking.

Core Tables & Their Purpose:

  • **Owner:** Stores details of vehicle owners.
  • **Vehicle:** Stores vehicle specifications and links to owners.
  • **Registration:** Manages registration status and renewals.
  • **Inspection:** Logs mandatory vehicle checks.
  • **Insurance:** Records policy details.
  • **History:** Maintains a searchable log of all major vehicle events.

Inspection Records

Demonstrating data from `Inspection` table, including failed inspections.

SQL Query:

-- Select all inspection records and filter for 'Failed'
SELECT V.registration_number, I.inspection_date, I.result
FROM Inspection I JOIN Vehicle V ON I.vehicle_id = V.vehicle_id
WHERE I.result = 'Failed';

Searchable Vehicle History

Illustrating a vehicle's event history from the `History` table.

SQL Query:

-- Select all history events for a specific vehicle (e.g., MH12 AB 1234)
SELECT V.registration_number, H.event_type, H.event_date, H.description
FROM History H JOIN Vehicle V ON H.vehicle_id = V.vehicle_id
WHERE V.registration_number = 'MH12 AB 1234'
ORDER BY H.event_date ASC;
